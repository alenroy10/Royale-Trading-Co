# Royale Trading Co. Website - AI Assistant Prompt

You are an expert AI web developer tasked with maintaining, updating, and extending the Royale Trading Co. website. This is a single-page marketing website for a premium green cardamom and spice export business based in Kerala, India.

## Tech Stack & Architecture
- **HTML5**: Semantic, accessible HTML.
- **CSS3**: Vanilla CSS (no Tailwind or other frameworks). The stylesheet is located at `css/styles.css`.
- **JavaScript**: Vanilla ES6+ JavaScript. The main script is located at `js/main.js`.
- **Build Tool**: Vite (configured in `vite.config.js`).

## Project Structure
- `index.html`: The main entry point and only HTML file.
- `css/styles.css`: Contains all custom styling.
- `js/main.js`: Contains all client-side interactivity (carousel, mobile nav, scroll effects, form submission).
- `public/`: Contains static assets like images, `robots.txt`, and `sitemap.xml` that are copied directly to the build output.
- `dist/`: The production build output directory (generated by running `npm run build`).

## Development Guidelines
1. **Aesthetics**: Maintain the premium, clean, and professional design aesthetic. Use existing CSS variables for colors, typography, and spacing.
2. **Performance**: Keep the site lightweight. Avoid adding heavy third-party libraries unless absolutely necessary.
3. **Responsiveness**: Ensure all new components and layouts are fully responsive across mobile, tablet, and desktop devices.
4. **Accessibility**: Use semantic HTML tags and appropriate ARIA attributes.
5. **No Frameworks**: Do not introduce React, Vue, Tailwind, or jQuery. Stick to vanilla web technologies.
6. **Production Build**: Before deploying, always run `npm run build` to generate the optimized assets in the `dist` folder.

## Key Features
- **Hero Carousel**: A custom auto-playing image slider with manual controls.
- **Scroll Reveal**: Elements fade in as they enter the viewport using `IntersectionObserver`.
- **Contact Form**: Submits data via FormSubmit to the business email address.

## Tasks
When requested to make changes, follow the guidelines above and ensure all code is production-ready, clean, and free of dead code.
